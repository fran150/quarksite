ko.validation Método

Permite activar validaciones sobre el observable. Se debe especificar el nombre a mostrar del campo (nombre con el que se muestra el campo al usuario) y la configuración de validación.

La configuración de validación consiste en un objeto donde cada propiedad es el nombre de un validador y el valor de la propiedad es la configuración que el validador necesita. Cuando se valida el observable Quark.js ejecutará todos los validadores activos en el observable pasandoles la configuración y si un validador detecta un valor invalido seteará las propiedades hasError y validationMessage con los detalles del error.

validation(displayName, validationConfig)

Parámetros

Nombre Tipo Descripción
displayName string Nombre del campo que se muestra al usuario. Se usa para armar el mensaje de error.
validationConfig Objeto Objeto donde cada propiedad es el nombre de un validador y el valor de la propiedad es la configuración que el validador necesita

Devuelve

El propio observable asi se puede definir las validaciones en la propia creación del observable. Por ejemplo: var validableObservable = ko.observable().validation({ required: { message: 'El campo no puede ser vacío' }, length: { min: 3, max: 20 } })
