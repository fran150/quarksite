Namespaces

Si bien Quark.js permite utilizar cualquier nombre de tag, es probable que en grandes aplicaciones la cantidad de componentes crezca rápidamente por lo que se hace primordial agruparlos y ordenarlos de alguna manera.

Quark.js permite agruparlos en "namespaces" para evitar el conflicto de nombres cuando existen muchos componentes permitiendo agrupar aquellos que estén relacionados entre sí bajo un mismo namespace.

El concepto de namespaces en si, es similar al de los directorios de archivos. Dos archivos se pueden llamar igual pero deben estar en distintos directorios. El separador de nombres en los componentes de Quark.js es el guion -. Luego Quark.js proporciona el binding namespace que permite crear un apodo o "alias" a un namespace mejorando la legibilidad del código fuente.

Por ejemplo, si se tienen componentes con los siguientes nombres:

docs-sidebar-main
docs-sidebar-ctrl-link
docs-sidebar-ctrl-button
docs-sidebar-ctrl-search
docs-navbar-main
docs-navbar-ctrl-link
docs-navbar-ctrl-button
docs-navbar-ctrl-search

Se pueden utilizar con namespaces de la siguiente manera: <quark-component> <div data-bind="namespace: 'docs-navbar', alias: 'nav'"> <div data-bind="namespace: 'docs-navbar-ctrl', alias: 'navc'"> <nav:main> <navc:link> <navc:button> <navc:search> </nav:main> </div> </div> <div data-bind="namespace: 'docs-sidebar', alias: 'sdb'"> <div data-bind="namespace: 'docs-sidebar-ctrl', alias: 'sdbc'"> <sdb:main> <sdbc:link> <sdbc:button> <sdbc:search> </sdb:main> </div> </div> </quark-component>

Existe una versión del método register que permite registrar los componentes ordenandolos por namespaces para facilitar la lectura de los mismos. Este método es onNamespace y permite encadenarlo con:

  • namespace (para definir un nuevo namespace) dentro del actual,
  • con register para registrar un componente dentro del namespace actual,
  • con endNamespace para volver al namespace anterior
Por ejemplo: $$.onNamespace('pages') .register('home', 'pages/home') .namespace('library') .register('finder', 'pages/library/finder') .register('details', 'pages/library/details') .endNamespace() .namespace('docs') .register('home', 'pages/docs/home') .register('installing', 'pages/docs/installing'); En este ejemplo se registran los componentes:
pages-home
pages-library-finder
pages-library-details
pages-docs-home
pages-docs-installing

Finalmente, tambien se puede utilizar la funcion registerComponents pasando un objeto donde cada propiedad es el nombre del namespace o tag y el valor de la propiedad es un string con el path al módulo web del componente o un objeto con esta misma estructura que define los componentes o namespaces hijos.

El ejemplo anterior se puede registrar también como:

$$.registerComponents({ "pages": { "home": "pages/home", "library": { "finder": "pages/library/finder", "details": "pages/library/details" }, "docs": { "home": "pages/docs/home", "installing": "pages/docs/installing" } } });
