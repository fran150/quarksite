outlet binding

Permite definir un outlet, las páginas pueden definir un conjunto de componentes que se deben mostrar en cada segmento, el outlet permite indicar donde se debe mostrar dicho segmento. Cada outlet tiene un nombre y se debe definir en la configuración de la página el nombre del outlet y el tag del componente que se debe mostrar, como puede verse en la definición del método routing.pages.

outlet: <name> [, resetScroll: <reset>]

Parámetros

Nombre Tipo Descripción
name string Nombre del outlet, debe corresponder con un outlet definido el algun segmento de la página.
reset boolean Indica si se debe resetear cualquier scroll realizado al cambiar el componente que se muestra en el outlet. Por defecto es true.

Observaciones

Si dos segmentos de la misma página definen distintos valores para el mismo outlet, Quark.js siempre mostrará el componente definido en el ultimo segmento.
