Modulos Web

Para definir un modulo web, utilizaremos la funcion define, el primer parametro debe ser un array con los nombres de las dependencias, y el segundo una función que se invocará una vez que se hayan cargado las mismas: define(['quark', 'knockout'], function($$, ko) { function ModuloDePrueba() { } return ModuloDePrueba; });

Este javascript define un modulo que depende de Quark.js y de Knockout.js y devuelve una funcion llamada ModuloDePrueba. Require.js realizará un request HTTP para cargar cada modulo y una vez que esten ambos cargados invocará a la funcion especificada pasando como parametros los modulos de las dependencias. Entonces en $$ estará el codigo de Quark.js y en ko el de Knockout.

Suponiendo que se guarda el modulo anterior dentro de la carpeta modulos y lo llamamos modulo.js se puede utilizar de dos maneras:

Como dependencia de otro modulo web: define(['quark', './modulos/Modulo'], function($$, modulo) { function OtroModulo() { var mod = new Modulo(); } return OtroModulo; }); En este ejemplo, se define otro modulo que tiene dos dependencias, Quark.js y el modulo anterior. Require.js cargará dichos modulos y una vez cargados invocará a la funcion pasando como parametros las dependencias cargadas. Dentro de la funcion se asigna a la variable mod un objeto de tipo ModuloDePrueba

Otra forma de uso es directamente dentro del código, utilizando la funcion require: . . require(['quark', './modulos/Modulo'], function($$, Modulo) { var mod = new Modulo(); }); . . El funcionamiento es similar al anterior; cargará Quark.js y ModuloDePrueba e invocará a la función pasando las dependencias como parametros. Dentro de la funcion se crea un objeto de tipo ModuloDePrueba y lo asigna en la variable mod, obteniendo el mismo resultado que en el punto anterior.

Como dependencia se puede especificar el path al archivo que contiene el modulo omitiendo la extensión js, o un alias. Los alias deben definirse en la configuración de require indicando a que path del servidor apuntan, ya sea un archivo especifico o una carpeta. Si quieres saber mas sobre esto consulta la sección Configuración de Quark.js

Web Components

Para definir un componente, se debe crear un modulo web que devuelva el resultado de la ejecución de la funcion component. Esta funcion recibe dos parametros, el modelo del componente que es una clase javascript que contiene la funcionalidad de dicho componente y un string con el HTML que define como se mostrará dicho componente en pantalla: define(['quark', 'knockout'], function($$, ko) { function ComponentExample(params, $scope, $imports) { var self = this; } return $$.component(ComponentExample, '<h1>Component Example</h1>'); });
