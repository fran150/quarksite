Controladores

Los controladores permiten enlazar y coordinar todos los componentes que se hayan definido en los segmentos de la pagina cargada en el navegador.

Un controlador es, básicamente, un modulo web que debe devolver la clase del controlador. Quark.js por defecto intenta cargar el controlador de un modulo buscándolo dentro de la carpeta controller del sitio y tomando el nombre de la ruta como path al modulo.

Por ejemplo, si la ruta es main/customer/editor, Quark.js intentará cargar 3 controladores:
controller/main.js
controller/main/customer.js
controller/main/customer/editor.js

Los componentes siguen la misma lógica que los componentes cuando se navega entre las paginas. Solo los controladores de los segmentos que difieren entre la pagina anterior y la nueva son modificados al cambiar de página.

Si desde el ejemplo anterior se navega a la ruta main/customer/viewer, Quark.js sólo intentará cargar el controlador controller/main/customer/viewer.js.

Si el controlador para un segmento especifico no se encuentra definido Quark.js crea uno por defecto.

Los componentes definidos en la página se importan automáticamente al controlador con el nombre del outlet, Quark.js envía como segundo parametro al controlador el tracker para acceder a estos modelos. El primer parámetro es una referencia al controlador padre.

Por ejemplo, teniendo en cuenta esta configuración: define(['quark'], function($$) { var config = { 'main': { navbar: 'navbar-component' }, 'main/home': { body: 'main-home' }, 'main/about': { body: 'main-about' }, 'docs': { navbar: 'docs-navbar-component' }, 'docs/home': { body: 'docs-home' }, 'docs/basics': { navbar: 'alt-navbar', body: 'docs-basics' } }; var params = { 'main/home': { language: 'es' }, 'main/about': [ 'param1', 'param2' ] } $$.routing.pages(config, params); }); Se puede definir un controlador en controller/main/home.js como: define(['quark', 'knockout'], function($$, ko) { function MainHomeController(parent, $imports) { var self = this; $$.wait($imports.ready, function() { var homeModel = $imports.get('body'); }); } }); En homeModel se almacenará el modelo del componente main-home

Se puede utilizar los controladores para definir los parametros que se pasan a los componentes que se definieron para los outlet del segmento al que corresponden. Para esto Quark.js busca si en el controlador hay definido un método sendParameters, si es así, lo ejecuta enviando el nombre del outlet y el nombre del tag del componente como parámetros, esta función debe devolver un objeto con el nombre y valor de los parámetros que se deben enviar el componente.

Por ejemplo, agregando la función al ejemplo anterior: define(['quark', 'knockout'], function($$, ko) { function MainHomeController(parent, $imports) { var self = this; $$.wait($imports.ready, function() { var homeModel = $imports.get('body'); }); this.color = ko.observable('red'); this.sendParameters = function(name) { switch(name) { case 'body': return { text: 'test', headerColor: self.color }; } } } }); Ahora se envía dos parámetros al componente main-home, text con el literal "test" y headerColor con el observable color del controlador. Esto permite utilizar variables y observables del controlador para controlar y coordinar los componentes de la página.

Adicionalmente el controlador cuenta con dos propiedades mas que Quark.js le crea por defecto.

  • outlets es un objeto con una propiedad por cada outlet definido en el segmento de la pagina correspondiente al controlador. El nombre de la propiedad es igual al nombre del outlet y el valor es el nombre del tag del componente que se está mostrando en dicho outlet
  • params es un objeto con una propiedad por cada parametro definido en el segmento de la página correspondiente al controlador. El nombre de la propiedad es igual al nombre del parametro de la pagina y la propiedad apunta al observable definido como parámetro.

La propiedad params se puede utilizar para reenviar los parametros de la ruta a un componente permitiendo que el componente tome el estado de la propia ruta.

define(['quark', 'knockout'], function($$, ko) { function MainHomeController(parent, $imports) { var self = this; $$.wait($imports.ready, function() { var homeModel = $imports.get('body'); }); this.color = ko.observable('red'); this.sendParameters = function(name) { switch(name) { case 'body': return { text: self.params['language'], headerColor: self.color }; } } } }); Aquí se modificó el ejemplo anterior para que como parametro text se envíe el parametro language de la ruta.
