Modulos Web

El bloque fundamental de Quark.js son los modulos web. Estos permiten encapsular una porción de código con una funcionalidad específica y exponerla para que otros módulos puedan utilizarla. Además permiten definir las dependencias con otros modulos web.

Todas las porciones de código en Quark.js se implementan como modulos web. Para la definición, administración y carga de estos modulos Quark utiliza Require.js. Además de permitir definir modulos web, Require carga de forma asincronica el modulo y sus dependencias en el navegador (esto se conoce como AMD). Puedes encontrar mas información acerca de porque utilizar modulos web y AMD en este articulo y este otro de la página de Require.

Para definir un modulo web, utilizaremos la funcion define, el primer parametro debe ser un array con los nombres de las dependencias, y el segundo una función que se invocará una vez que se hayan cargado las mismas: define(['quark', 'knockout'], function($$, ko) { function ModuloDePrueba() { } return ModuloDePrueba; });

Este javascript define un modulo que depende de Quark.js y de Knockout.js y devuelve una funcion llamada ModuloDePrueba. Require.js realizará un request HTTP para cargar cada modulo y una vez que esten ambos cargados invocará a la funcion especificada pasando como parametros los modulos de las dependencias. Entonces en $$ estará el codigo de Quark.js y en ko el de Knockout.

Suponiendo que se guarda el modulo anterior dentro de la carpeta modulos y lo llamamos modulo.js se puede utilizar de dos maneras:

Como dependencia de otro modulo web: define(['quark', './modulos/Modulo'], function($$, modulo) { function OtroModulo() { var mod = new Modulo(); } return OtroModulo; }); En este ejemplo, se asigna a la variable mod un objeto de tipo ModuloDePrueba

Utilizando la funcion require: . . require(['quark', './modulos/Modulo'], function($$, Modulo) { var mod = new Modulo(); }); . . Esta funcion cargará Quark.js y ModuloDePrueba y una vez cargados invocará la funcion pasando las dependencias como parametros. En esta funcion se crea un objeto de tipo ModuloDePrueba y lo asigna en la variable mod, obteniendo el mismo resultado que en el punto anterior.

Como dependencias se puede especificar el path al archivo que contiene el modulo omitiendo la extensión js, o un alias. Los alias deben definirse en la configuración de require indicando a que path del servidor apuntan, ya sea un archivo especifico o una carpeta. Si quieres saber mas sobre esto consulta la sección Configuración de Quark.js
