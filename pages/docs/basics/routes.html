Rutas

Las rutas permiten asociar un hash de la URL y la página de Quark.js que se desea mostrar cuando se dirija el navegador a dicha URL. Para el sistema de ruteo, Quark.js utiliza Crossroads.js y Hasher.

Mediante las rutas el programador puede definir la url que utilizará el usuario para acceder a una pagina especifica y que valor setear para los parametros de dicha pagina. De esta forma, mediante la URL se podrá controlar el estado de la aplicación. Gracias a Hasher.js el navegador almacena en su historial las diferentes rutas que fue visitando por lo que el usuario podrá navegar por las distintas rutas incluso utilizando los botones adelante y atrás del navegador.

En la documentación de Crossroads.js se indica la estructura que pueden tener las rutas y como definir variables en las mismas. Si el nombre de la variable de la ruta coincide con un parametro de la pagina asociada, se actualiza el observable de la pagina con el valor especificado en la url para dicha variable.

Por ejemplo, si la url base de tu sitio es http://example.com y defines las rutas asi: var routes = { 'main/home': 'home', 'main/components/edit': 'edit/{id}' }

Ingresando a la URL http://example.com#home va a cargar la pagina main/home, cambiando la URL a http://example.com#edit/5 cargará la pagina main/components/edit y si tiene un parametro id le asignará el valor 5. Luego si la url cambia a por ejemplo a http://example.com#edit/7 no se cargará de nuevo la pagina, ya que el hash coincide con el de la ruta actual; sólo se actualizará el valor del parametro id a 7.
