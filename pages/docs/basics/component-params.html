Parametros de Componentes

Asi como los elementos HTML pueden tener atributos que modifican su comportamiento o apariencia los componentes pueden definir parámetros.

Los parámetros se definen en el modelo del componente utilizando la funcion parameters de Quark.js. Esta funcion se debe invocar con:

  • Un objeto donde cada propiedad es el nombre y valor inicial de un parametro que aceptará el componente.
  • Como se especificó en el apartado de Componentes Web, Quark.js envia al modelo 3 parámetros, el primero de ellos es un objeto con el una propiedad por con el nombre y valor de cada parametro que se pasa al componente.
  • El objeto o una matriz de objetos en donde se deben crear las propiedades que corresponden con cada parametro que el componente acepte.

Los parametros pueden ser observables, o literales de Javascript. Por ejemplo: define(['quark', 'knockout', 'text!./example.html'], function($$, ko, template) { function ComponentExample(params, $scope, $imports) { var self = this; $$.parameters({ obs: ko.observable('Hola'), str: 'Mundo', num: 2016 }, params, this); } return $$.component(ComponentExample, template); }); define 3 parametros:

  1. obs que es un observable con valor inicial Hola
  2. str que es un string con el valor inicial Mundo
  3. num que es un numero con el valor inicial 2016

Si se registró este componente bajo el tag component-example el que lo utilice podrá pasarle valores customizados a los parametros de la siguiente forma: <component-example params="obs: 'Adios', str: 'Planeta', num: 2015"> </component-example>
