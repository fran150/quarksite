Páginas

Las paginas en Quark.js permiten definir un conjunto de componentes que deben mostrarse en un momento dado al usuario.

En la vista de cualquier componente se puede usar el binding outlet: <nombre> para especificar el lugar donde se debe mostrar un componente va a ser definido por la pagina. Por ejemplo: <div data-bind="outlet: 'navbar'"> </div> <div data-bind="outlet: 'body'"> </div> Aquí se usan dos div en donde se va a mostrar el componente que se defina en la página para los outlets navbar y body

Las paginas se definen en segmentos organizados jerarquicamente. Cada segmento puede definir el nombre de uno o mas outlets y que componente deben mostrarse en los mismos. Al cambiar de pagina solo se actualizan los componentes definidos en los segmentos que no son compartidos por ambas paginas. Siguiendo el ejemplo anterior, supongamos que definimos, las paginas como:

var pages = { 'main': { navbar: 'navbar-component' }, 'main/home': { body: 'main-home' }, 'main/about': { body: 'main-about' }, 'docs': { navbar: 'docs-navbar-component' }, 'docs/home': { body: 'docs-home' }, 'docs/basics': { navbar: 'alt-navbar', body: 'docs-basics' } }

  • Al mostrar la pagina main/home:
    • en el div navbar se mostrará el componente navbar-component
    • en el div body se mostrará el componente main-home
  • Si se cambia la pagina por main/about:
    • solo se actualizará el div body mostrando el componente main-about
  • Al cambiar a docs/home se cambiaran ambos componentes:
    • en el div navbar se mostrará el componente docs-navbar-component
    • en el div body se mostrará el componente docs-home
  • Finalmente, los outlet definidos en segmentos hijos tienen mayor prioridad que los definidos en segmentos superiores, por lo que al navegar a la pagina docs/basics se modificarán ambos div mostrando:
    • en el div navbar se mostrará el componente alt-navbar
    • en el div body se mostrará el componente docs-basics

Las paginas pueden definir parametros que Quark.js convertirá en observables cuyos valores se obtendrán de la URL definida en la ruta como veremos a continuación.
